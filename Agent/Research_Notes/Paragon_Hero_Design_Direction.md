# Paragon 英雄设计方向

生成日期：2026-07-14

## 目标

Apex 第一阶段以 Paragon 资产包作为英雄、地图、动画和技能表现的验证基础。我们不再先抽象设计一堆脱离资产的技能，而是先看角色动作、Montage、Socket、VFX 和地图空间，再设计适合这些资源的英雄与技能。

## 当前决定

- 项目形态：第三人称动作 / 英雄技能原型。
- 视角方向：鼠标控制摄像机，后续可以加入屏幕中心准星。
- 资产方向：优先使用 Paragon 角色、地图、动画、Montage、特效。
- 技能设计方法：根据 Montage 和动作语义反推技能，而不是硬找动画匹配技能。

## 设计原则

1. 资产先行：每个技能候选都要能说明它依赖哪个角色动作、哪个 Montage 或哪类动画语义。
2. 一次只验证一个英雄：先让一个 Paragon 角色完成移动、镜头、动画和一个技能闭环，再扩展到其他英雄。
3. 不急着重定向泛化：除非一个技能框架已经跑通，否则不要批量处理角色重定向和动画替换。
4. MCP 不负责单个编辑器配置：角色替换、AnimClass、GameMode、单个 Notify 等操作优先给人工步骤。
5. 技能系统仍保持通用目标：Paragon 只是验证素材，不把框架写死成某个英雄专用系统。

## 第三人称基线

第一阶段先建立可靠的第三人称体验：

- `Move`：WASD 按摄像机方向移动。
- `Look`：鼠标 X/Y 控制摄像机 yaw / pitch。
- `Camera`：使用 SpringArm + FollowCamera 的第三人称结构。
- `Rotation`：先明确移动朝向和镜头朝向的关系，技能瞄准再单独讨论。
- `Input`：使用 Enhanced Input，后续技能输入也接入同一套 Mapping Context。

## Montage 驱动技能流程

每个候选英雄先做一份 Montage 目录：

| 项 | 需要记录 |
| --- | --- |
| Montage 名称 | 资产路径、所属角色、Slot |
| 动作语义 | 施法、挥砍、射击、蓄力、跳跃、受击等 |
| 关键帧 | 哪一帧适合发射、命中、生成范围、结束引导 |
| Notify | 是否已有 Notify，是否需要 GameplayEvent Notify |
| Socket | 手、武器、头、地面点、投射物出生点 |
| 技能候选 | 可映射成什么技能原型 |

## 下一步

1. 用户确认 Paragon 资产是否已经迁移到 Apex，或者仍在 Vault / 其他项目中。
2. 先选择一个角色作为第一名验证英雄。
3. 建立该角色的基础移动和第三人称鼠标视角。
4. 盘点该角色 Montage，并从动作语义反推第一批技能候选。
