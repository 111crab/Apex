# 协作约定

*最后更新：2026-07-07*

## 目的

Aura 项目将从一个 UE C++ / GAS 学习项目，逐步发展为一个可扩展的 GAS 技能系统，并在后续阶段设计技能编辑器。所有重要工作都应该先规划、再执行、再审查，避免多个 agent 同时把项目推向不可控状态。

## 角色分工

**用户**
- 拥有最终产品方向和最终审批权。
- 审核关键规划、架构选择和阶段成果。
- 当 UE 编辑器、MCP、资产或设计问题需要人工判断时介入。

**Codex**
- 负责高层规划、架构讨论、审查、文档维护。
- 为 Claude 编写具体实施 Prompt。
- 审查 Claude 的代码、报告和 MCP/编辑器操作结果。
- 维护任务池、决策记录、项目上下文、阶段总结。

**Claude**
- 负责执行已经批准的具体实施任务。
- 可以写代码、操作 UE MCP/编辑器，但必须遵守 Prompt 范围。
- 每次实施后必须把报告写入 `Agent/04_AgentHandoff/Claude_Reports/`。

## 规划规则

重大实现开始前，必须满足：
- 阶段目标已经写清楚。
- 预期产出已经写清楚。
- Codex 已经写出计划或 Claude Prompt。
- 如果任务涉及架构、代码组织、资产、编辑器工作流，需要用户先确认方向。

小范围文档补充、事实盘点、审计报告可以直接执行。

## 交接规则

Codex 写给 Claude 的 Prompt 应该包含：
- 目标。
- 范围。
- 推荐阅读的文件。
- 允许修改的文件。
- 建议使用的 Skill / MCP。
- 必须输出的报告路径。
- 验证方式。
- 明确禁止做的事情。

Claude 的报告应该包含：
- 简要总结。
- 修改了哪些文件。
- 运行了哪些命令或测试。
- 使用了哪些 UE 编辑器/MCP 操作。
- 已知问题。
- 后续建议。

## 审查规则

Codex 审查 Claude 产出时重点看：
- 是否符合已批准的 Prompt。
- 是否符合 GAS 和 UE 生命周期。
- Runtime 模块和 Editor 模块边界是否清晰。
- 是否考虑复制、预测、客户端/服务器职责。
- 是否有资产或配置副作用。
- 验证是否足够。

## 文档语言规则

需要用户审核、讨论、决策的文档尽量使用中文。  
主要给工具或 skill 读取的结构化上下文可以保留英文，但如果用户需要审核，应补充中文说明。

## 代码注释与学习文档规则

Aura 既是项目，也是 UE C++ / GAS 学习载体。正式写代码时，实施 agent 必须让代码和文档帮助用户理解系统，而不是只交付能运行的结果。

代码注释要求：
- 对非显然的 GAS 生命周期、ASC 所有权、预测、复制、Commit、取消、GameplayEvent、GameplayCue、EffectContext、Attribute 结算等逻辑写解释性注释。
- 对解耦边界写简短说明，例如“为什么这里用配置资产而不是硬编码技能类”“为什么表现层不决定权威伤害”。
- 避免噪音注释，不解释显而易见的语法或单行赋值。
- 如果代码里存在为了教程兼容、迁移过渡、多人同步或编辑器未来扩展而做的折中，必须注释说明原因。
- 如果设计思想放进代码注释会让代码变啰嗦，应改写到同目录或 `Agent/` 下的中文 MD 文档中，并在实现报告里指出阅读入口。

文档要求：
- 每引入一个新的核心概念、系统边界或可复用模式，都应该补一段中文说明文档。
- Claude 的实施报告不仅要列出改动，还要解释关键实现思路和用户后续应该阅读的文件。
- Codex 写给 Claude 的 Prompt 必须显式要求“学习型注释”和“中文实现说明”。

直接添加 C++ 文件要求：
- 可以直接新增 `.h` / `.cpp` 文件，不强制通过 UE 编辑器向导。
- 新增 Runtime 文件应放在 `Source/Aura/Public` 或 `Source/Aura/Private`，Editor-only 代码应进入 Editor 模块或插件。
- 新增反射类型必须正确使用 `.generated.h`、UE 宏和模块依赖。
- 实施报告必须说明新增文件路径、模块归属、是否修改 `Aura.Build.cs`，以及是否需要重新生成项目文件/刷新 IDE/重启编辑器。

## 文档规则

- 长期决策写入 `Agent/00_Coordination/Decision_Log.md`。
- Prompt 和实施报告写入 `Agent/04_AgentHandoff/`。
- UE 项目事实写入 `.agents/ue-project-context.md`，供 UE skills 自动参考。

## 范围边界

- 不在没有迁移计划的情况下整体重写教程代码。
- 不把 editor-only 依赖塞进 `Aura` Runtime 模块。
- 不为每一种技能机制都单独创建一套互不复用的系统。
- 技能系统优先考虑“少量 C++ 运行时能力 + 数据配置资产”的方向，为后续技能编辑器服务。
- `Agent/UE_GAS_SkillGraph_Design_Document.md` 暂时只作为未评审参考。
