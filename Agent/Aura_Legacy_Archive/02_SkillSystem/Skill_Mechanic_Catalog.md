# 技能机制总目录

*创建日期：2026-06-26*  
*状态：Phase 1 已确认目录。后续如发现遗漏，可增量补充；本文件不是实现承诺。*

## 目的

之前的 `Skill_Mechanic_Taxonomy.md` 只确认了第一批原型，这不等于我们已经完成“技能机制总览”。

本文件系统整理市面常见游戏里值得抽象的技能机制。后续我们再基于它讨论：
- 哪些机制应该第一批实现。
- 哪些机制第二批、第三批实现。
- 哪些机制只需要保留扩展点，不需要 V1 实现。
- 哪些机制对 GAS、动画、表现、UI、网络同步、编辑器设计有特殊要求。

这里的“技能机制”不是具体技能名，而是可复用的玩法结构。例如“火球术”不是一个机制，“投射物 + 命中后伤害 + 爆炸表现”才是机制组合。

## 总体分层

一个技能通常可以拆成这些层：

| 层 | 关注问题 | 例子 |
| --- | --- | --- |
| 激活 | 技能如何开始 | 按键、蓄力、被动触发、连招输入 |
| 目标 | 技能作用于谁/哪里 | 自身、敌人、地面点、范围、路径、链式目标 |
| 传递 | 效果如何抵达目标 | 直接应用、投射物、光束、近战 Trace、区域 Actor |
| 效果 | 对玩法状态产生什么改变 | 伤害、治疗、Buff、控制、召唤、位移 |
| 时间 | 效果如何随时间展开 | 瞬发、延迟、持续、周期、阶段、开关 |
| 资源 | 技能消耗或限制什么 | 法力、冷却、能量、弹药、层数、生命 |
| 交互 | 和其他技能/状态如何组合 | 叠层、引爆、驱散、免疫、反制、连携 |
| 表现 | 玩家看到/听到什么 | 动画、Niagara、音效、UI、镜头 |
| 同步 | 多人下谁说了算 | 预测、服务器结算、复制、Cue、回滚 |

这套拆法很重要，因为“市面上所有技能”并不是靠枚举每个技能名覆盖，而是靠这些层的组合覆盖。

## 1. 激活机制

| 机制 | 说明 | 典型例子 | GAS/UE 关注点 |
| --- | --- | --- | --- |
| 瞬发主动 | 按下立即结算 | 治疗术、闪电击 | `CommitAbility`、目标验证、即时 GE |
| 前摇施法 | 激活后先播放施法/挥砍，再结算 | 火球、重击 | Montage、施法取消、Commit 时机 |
| 持续引导 | 按住或激活后持续生效，打断则停止 | 光束、持续治疗 | AbilityTask、周期消耗、取消清理 |
| 蓄力释放 | 按住积累强度，松开释放 | 蓄力箭、蓄力法术 | 输入按下/松开、蓄力曲线、预测边界 |
| 开关型 | 开一次开启，再开一次关闭 | 光环、持续护盾 | Infinite GE、状态 Tag、关闭清理 |
| 被动常驻 | 不需要主动按键，一直监听条件 | 加成被动、受击反弹 | 监听事件、内部冷却、防递归 |
| 条件触发 | 条件满足时自动触发 | 暴击回血、击杀刷新 | GameplayEvent、Tag 条件、Proc Chance |
| 多阶段/连招 | 多次输入推进阶段 | 三段斩、格斗连段 | Combo State、输入窗口、Montage Section |
| 二次释放/Recast | 首次释放后可再次触发后续效果 | 标记后引爆、回旋飞刃收回 | Active Ability 状态、Reactivation 规则 |
| 姿态/模式 | 切换技能组或角色规则 | 战斗姿态、元素形态 | 授予/移除 Ability、Tag 阻断 |
| 大招/终极技 | 强限制高收益技能 | 终极爆发、全屏技能 | 资源、冷却、表现、网络带宽 |

## 2. 目标选择机制

| 机制 | 说明 | 典型例子 | GAS/UE 关注点 |
| --- | --- | --- | --- |
| 自身 | 只作用于施法者 | 自疗、护盾 | Self Target、GE to Self |
| 单体目标 | 选择一个角色或对象 | 点名治疗、斩杀 | 目标合法性、阵营过滤 |
| 地面点 | 选择世界坐标 | 暴风雪、传送门 | 鼠标拾取、Nav/地面验证 |
| 范围目标 | 半径、盒子、扇形、圆环等区域 | 震荡波、范围治疗 | Overlap / Sweep / Query |
| 路径目标 | 沿线、曲线、弹道路径 | 穿透箭、火墙 | Trace、路径采样、阻挡 |
| 方向技能 | 朝输入方向释放，不锁定目标 | 技能射击、冲击波 | 客户端输入、服务器验证 |
| 锁定目标 | 锁定后持续追踪 | 追踪飞弹、锁链 | Target Retention、失效处理 |
| 链式目标 | 命中后跳转多个目标 | 链式闪电 | 候选排序、防重复、衰减 |
| 随机/规则选择 | 从候选中按规则选目标 | 随机箭雨、优先低血量 | 服务器确定随机、可复现 |
| 全局/全队 | 作用于全地图或队伍 | 全队 Buff、全屏伤害 | 目标集合、性能、可见性 |

## 3. 传递与载体机制

| 机制 | 说明 | 典型例子 | GAS/UE 关注点 |
| --- | --- | --- | --- |
| 直接应用 | 目标确定后立即应用效果 | 治疗、点名伤害 | GE Spec、Target ASC |
| 投射物 | Actor 飞行后命中 | 火球、箭矢 | 服务器生成、命中权威、复制表现 |
| Hitscan | 瞬时射线命中 | 枪械、激光点射 | Trace、命中校验、延迟补偿可后置 |
| 近战 Trace | 动画窗口内扫掠检测 | 普攻、连招 | Montage Notify、Trace、重复命中控制 |
| 光束/Tether | 持续连接目标或方向 | 激光、吸血链 | 目标刷新、断线、持续 Cue |
| 区域 Actor | 世界中存在一段时间 | 火墙、毒云、治疗圈 | 生命周期、周期、进入/离开事件 |
| 附着效果 | 附在角色或物体上 | 燃烧、护盾球 | Attach、Cue 生命周期 |
| 召唤物 | 生成有行为的单位 | 宠物、幻象 | 所有权、AI、ASC、复制 |
| 陷阱/地雷 | 放置后等待触发 | 地雷、夹子 | 触发条件、可见性、过期清理 |
| 屏障/实体 | 创建阻挡或可交互实体 | 冰墙、护盾墙 | Collision、生命值、可破坏 |

## 4. 效果机制

| 机制 | 说明 | 典型例子 | GAS/UE 关注点 |
| --- | --- | --- | --- |
| 直接伤害 | 立即扣血 | 火球命中 | ExecCalc、抗性、暴击、格挡 |
| 直接治疗 | 立即回血 | 治疗术 | Instant GE、最大生命 Clamp |
| 持续伤害 DoT | 一段时间周期扣血 | 中毒、燃烧 | Duration/Periodic GE |
| 持续治疗 HoT | 一段时间周期回血 | 回复术 | Periodic GE、叠加规则 |
| 属性 Buff | 提升属性或战斗能力 | 加速、增伤 | Duration/Infinite GE、Stack |
| 属性 Debuff | 降低属性或施加弱化 | 虚弱、破甲 | Tag、Stack、免疫 |
| 护盾/吸收 | 先消耗护盾再扣血 | 魔法盾 | Shield Attribute 或伤害路由 |
| 免疫/无敌 | 忽略部分或全部效果 | 无敌、魔免 | Tag Gate、GameplayEffect Immunity |
| 反射/反弹 | 把伤害或效果返还来源 | 反伤甲 | 事件监听、防递归 |
| 吸血/资源偷取 | 造成伤害后恢复资源 | 生命偷取 | Source/Target 上下文 |
| 控制效果 | 限制行动或输入 | 眩晕、沉默、定身、缴械 | Tag 阻断、输入/移动/Ability Block |
| 软控制 | 影响但不完全禁止行动 | 减速、致盲、嘲讽、恐惧 | 行为系统、移动、AI |
| 清除/驱散 | 移除 Buff 或 Debuff | 净化、驱散 | Remove GE by Tag/Handle |
| 复制/偷取 Buff | 转移或复制状态 | 偷 Buff、传播诅咒 | Active GE 查询与重建 |
| 标记/层数 | 给目标叠加可引爆状态 | 易伤标记、连击点 | Stack、Tag、SetByCaller |
| 引爆/消耗层数 | 消耗已有标记造成效果 | 爆破印记 | 查询 Stack、清理时机 |
| 斩杀/阈值效果 | 根据生命或状态增强 | 低血斩杀 | Attribute 查询、条件倍率 |
| 复活/死亡保护 | 死亡后恢复或阻止死亡 | 复活、免死 | 死亡流程、状态机 |
| 变形/变身 | 改变角色能力与表现 | 熊形态、恶魔化 | Mesh/Anim/Ability Set 切换 |
| 资源操作 | 改变法力、能量、怒气等 | 回蓝、烧蓝 | Attribute、资源规则 |
| 冷却操作 | 缩短、刷新或延长冷却 | 击杀刷新 | Cooldown GE / Tag 管理 |

## 5. 位移与空间机制

第一批不做位移，但完整目录需要保留它们，因为很多游戏技能离不开空间操控。

| 机制 | 说明 | 典型例子 | GAS/UE 关注点 |
| --- | --- | --- | --- |
| 冲刺 | 快速移动一段距离 | 翻滚、突进 | CharacterMovement、预测、碰撞 |
| 传送 | 瞬间改变位置 | 闪现 | 目标验证、穿墙、服务器校验 |
| 击退/牵引 | 改变目标位置 | 击飞、拉人 | Launch/RootMotion、网络修正 |
| 跳跃/跃击 | 位移加落点效果 | 跳斩 | Motion Warping、落点验证 |
| 交换位置 | 两个单位互换 | 换位术 | 权威位置、碰撞处理 |
| 聚怪/推开 | 范围空间控制 | 黑洞、冲击波 | 多目标移动、性能 |
| 地形/区域改变 | 创建阻挡或危险区域 | 冰墙、火墙 | Collision、Nav、生命周期 |

## 6. 时间、资源与限制机制

| 机制 | 说明 | 典型例子 | GAS/UE 关注点 |
| --- | --- | --- | --- |
| 冷却 | 使用后等待恢复 | 大多数技能 | Cooldown GE、UI 查询 |
| 公共冷却 | 一组技能共享短 CD | MMO GCD | 共享 Tag、阻断规则 |
| 充能次数 | 技能有多次存储次数 | 三层闪避 | Charge Stack、恢复计时 |
| 弹药/弹夹 | 使用消耗弹药 | 射击技能 | Inventory/Attribute、装填 |
| 法力/能量/怒气 | 消耗不同资源 | 法术、战士技能 | Cost GE、资源 Attribute |
| 生命消耗 | 用生命施法 | 血魔法 | 成本合法性、死亡边界 |
| 持续消耗 | 引导或开关时持续扣资源 | 持续护盾 | Periodic Cost、取消 |
| 施法时间 | 技能完成前可被打断 | 读条火球 | Cast State、打断事件 |
| 延迟生效 | 释放后过一段时间生效 | 延迟爆炸 | Timer、Cue、取消 |
| 内部冷却 | 被动触发后短时间不再触发 | 暴击回血 | Internal Cooldown Tag/GE |
| 随机触发 | 有概率发生 | 触发闪电 | 服务器随机、可调试 |

## 7. 组合、反制与规则机制

| 机制 | 说明 | 典型例子 | GAS/UE 关注点 |
| --- | --- | --- | --- |
| 元素克制 | 不同元素有抗性/弱点 | 火/冰/雷 | Damage Type Tag、Resistance |
| 状态反应 | 两种状态组合产生新效果 | 冰冻后碎裂 | Tag Query、事件链 |
| 连携技能 | 一个技能改变另一个技能 | 标记后强化攻击 | Ability Set、Tag 条件 |
| 反制/打断 | 阻止对方施法或技能 | 沉默、打断 | Cancel Ability、Tag Block |
| 免疫与抗性 | 对某些效果无效或减弱 | Boss 免控 | Immunity Tag、DR 规则 |
| 递减控制 | 重复控制时间递减 | PvP 控制衰减 | Stack/分类/时间窗口 |
| 阵营规则 | 区分友方/敌方/中立 | 治疗友军 | Team/Faction 过滤 |
| 仇恨/威胁 | 影响 AI 攻击目标 | 嘲讽 | AI 感知/黑板/Tag |
| 条件倍率 | 根据状态变化数值 | 背刺、低血增伤 | SetByCaller、ExecCalc |
| 共享/传播 | 效果传播给附近单位 | 瘟疫传播 | 目标查询、防无限传播 |

## 8. 表现与反馈机制

表现不是玩法结算，但它决定技能是否“像一个技能”。

| 机制 | 说明 | 典型例子 | GAS/UE 关注点 |
| --- | --- | --- | --- |
| 施法动画 | 开始、循环、结束动画 | 法师施法 | Montage、AnimNotify |
| 命中表现 | 命中时特效和音效 | 爆炸、受击 | GameplayCue |
| 持续表现 | 持续存在的视觉/音效 | 火圈、光环 | Cue Add/Remove |
| UI 冷却 | 显示剩余冷却和可用性 | 技能栏 | ASC 查询 Cooldown Tag |
| 资源不足反馈 | 告诉玩家为什么不能放 | 法力不足 | 失败原因、UI Message |
| 范围预览 | 释放前显示范围 | AOE 圆圈 | Decal/Widget/Debug |
| 目标提示 | 高亮可选目标 | 锁定边框 | Targeting UI |
| 镜头/震动 | 强化打击感 | 大招震屏 | 本地表现，不改玩法 |
| 浮动文字 | 伤害、治疗、暴击 | Damage Text | 当前项目已有基础 |

## 9. 多人同步关注点

每类技能都要回答这些问题：

| 问题 | 说明 |
| --- | --- |
| 谁请求激活 | 玩家客户端、AI 服务器、被动事件 |
| 谁做权威结算 | 通常是服务器 |
| 哪些状态必须同步 | Attribute、GE、Tag、关键 Actor |
| 哪些只是表现 | Niagara、音效、镜头、本地 UI |
| 是否需要预测 | 输入响应、投射物表现、近战手感 |
| 是否需要回滚 | 客户端预测被服务器拒绝 |
| 是否支持迟加入 | 持续 Buff、区域、召唤物是否能恢复当前状态 |
| Listen Server 是否重复执行 | Host 同时是服务器和本地玩家 |

## 10. 按复杂度的建议分期

这是初步分期，不是最终承诺。

| 阶段 | 推荐机制 | 目的 |
| --- | --- | --- |
| V1-A 基础闭环 | 瞬发治疗/伤害、火球投射物、范围持续地面技能、蓄力射击、三段连招、受击反弹被动 | 验证通用运行时主干 |
| V1-B 状态扩展 | Buff/Debuff、叠层、驱散、护盾、控制、开关光环 | 验证 GE/Tag/Stack 体系 |
| V1-C 载体扩展 | 陷阱、召唤物、光束、链式技能、Hitscan | 验证 Actor 生命周期和多种 Delivery |
| V2 空间机制 | 冲刺、传送、击退、牵引、跳斩、聚怪 | 验证 Movement/Prediction/Collision |
| V3 复杂组合 | 状态反应、元素连携、技能变体、姿态、复活、复制/偷取 Buff | 验证高层规则组合 |
| V4 编辑器深化 | 图形化编排、校验、预览、调试、版本管理 | 在运行时语义稳定后建设技能编辑器 |

## 对当前第一批的定位

第一批不是“技能系统全部能力”，而是从总目录中挑出的基础压力测试组合。它们应该覆盖：
- 一个最简单的直接效果。
- 一个移动载体投射物。
- 一个持续世界区域。
- 一个输入时序技能。
- 一个动画/近战窗口技能。
- 一个被动触发技能。

如果这 6 个做得足够干净，后续很多复杂技能都能变成这些机制的组合或扩展。
